Put your book info, images, links, whatever here. This box will sit on the right on desktop, and stack under the calendar on mobile.

If your right side was getting squished, it’s usually an overflow culprit (images/iframes/100vw). The CSS included below handles the usual crimes.

/* PASTE THIS IN YOUR EXTERNAL CSS FILE (e.g., styles.css) ------------------------------------------------------ If you already have a stylesheet, just add these rules. */ /* 1) Universal sanity: fixes a lot of mysterious “why is the right side broken” behavior */ *, *::before, *::after { box-sizing: border-box; } html, body { width: 100%; max-width: 100%; margin: 0; padding: 0; overflow-x: hidden; /* prevents a single rogue element from forcing horizontal overflow */ } /* 2) Page wrapper: adjust max-width if you want wider */ .page { width: 100%; max-width: 1100px; margin: 0 auto; padding: 20px 16px; } .page-title { margin: 0 0 16px; } /* 3) Desktop layout: two columns */ .two-col { display: flex; align-items: flex-start; gap: 24px; } /* CRITICAL: allows flex children to shrink instead of overflowing and squishing the right edge */ .two-col .panel { min-width: 0; } /* Left column (calendar) */ .calendar-panel { flex: 1; } /* Right column (book) */ .book-panel { flex: 1.3; } /* 4) Panel styling (optional, delete if you hate it) */ .panel { border: 2px solid #333; padding: 16px; background: #000; color: #fff; } .panel-title { margin: 0 0 12px; } /* 5) Embed handling: iframes/images often cause overflow */ img, iframe, video { max-width: 100%; height: auto; } /* Calendar iframe should never force a fixed width */ .embed-wrap { width: 100%; } .embed-wrap iframe { width: 100%; max-width: 100%; height: 600px; /* adjust */ border: 0; display: block; } /* 6) Mobile: stack */ @media (max-width: 700px) { .two-col { flex-direction: column; } .calendar-panel, .book-panel { width: 100%; } .embed-wrap iframe { height: 500px; /* adjust */ } }
